# Синхронизация с Supabase

## Обзор

LifeOS использует **offline-first** архитектуру с синхронизацией между локальной базой данных (IndexedDB) и облачной базой данных (Supabase PostgreSQL).

## Архитектура

```
┌─────────────────┐         ┌─────────────────┐
│   IndexedDB     │◄───────►│    Supabase     │
│   (Dexie.js)    │  Sync   │  (PostgreSQL)   │
│   Локальная БД  │  Service│   Облачная БД   │
└─────────────────┘         └─────────────────┘
       ▲
       │
   CRUD операции
       │
       ▼
┌─────────────────┐
│  React Query    │
│    Хуки         │
└─────────────────┘
```

## Как это работает

### 1. Создание/Обновление записей

Все CRUD операции работают **локально** через IndexedDB:

```typescript
// Создание транзакции
const createTransaction = useCreateTransaction();
createTransaction.mutate({
  account_id: '123',
  amount: 1000,
  type: 'expense',
  // ... другие поля
});
```

- Запись сохраняется в локальную базу данных (Dexie.js)
- Автоматически получает `sync_status: 'local'`
- Запускается фоновая синхронизация с Supabase

### 2. Синхронизация

**Автоматическая синхронизация:**
- Запускается каждые 30 секунд
- Срабатывает при появлении интернет-соединения
- Инициализируется при старте приложения

**Ручная синхронизация:**
```typescript
import { useSync } from '@/shared/hooks';

const { sync, isSyncing, error } = useSync();
await sync();
```

### 3. Статусы синхронизации

| Статус | Описание |
|--------|----------|
| `local` | Запись создана/изменена локально, ожидает синхронизации |
| `synced` | Запись синхронизирована с сервером |
| `conflict` | Обнаружен конфликт версий |
| `pending` | Запись в очереди на синхронизацию |

## Компоненты

### SyncStatus

Индикатор статуса синхронизации в sidebar:

- 🟢 Зеленый индикатор — онлайн
- 🔴 Красный индикатор — оффлайн
- 🔄 Анимация — идет синхронизация
- ⏱️ Время последней синхронизации
- 🔄 Кнопка ручной синхронизации

### Хуки

```typescript
// Статус синхронизации
const { isSyncing, lastSyncTime, lastSyncResult } = useSyncStatus();

// Ручная синхронизация
const { sync, isSyncing, error, lastResult } = useSync();

// Онлайн/оффлайн статус
const { isOnline, isOffline } = useOnlineStatus();
```

## Принцип работы

### Push (локаль → сервер)

1. Получение записей со статусом `local`
2. Проверка существования записи на сервере
3. Создание или обновление записи на сервере
4. Обновление статуса на `synced`

### Pull (сервер → локаль)

1. Получение всех записей пользователя с сервера
2. Сравнение версий записей
3. Обновление локальных записей (если remote версия новее)
4. Пометка конфликтов (если local версия новее)

## Конфликты

Конфликт возникает, когда:
- Одна и та же запись изменена локально и на сервере
- Версии записей не совпадают

**Разрешение конфликтов:**
- Локальная версия помечается как `conflict`
- Требуется ручное вмешательство (будущая функциональность)

## Offline режим

Приложение **полностью работает offline**:

✅ Все CRUD операции доступны  
✅ Данные сохраняются в IndexedDB  
✅ Интерфейс отображает оффлайн-статус  
✅ При появлении соединения — автоматическая синхронизация

## Ограничения

- Синхронизация работает только для авторизованных пользователей
- Конфликты версий пока не разрешаются автоматически
- Первая загрузка требует подключения к интернету
